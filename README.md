## 🎯 Overview

RecipeEasy is a modern, AI-powered recipe generation platform that leverages cutting-edge language models and image generation AI to create personalized cooking experiences. Built with Next.js 14 and Cloudflare Workers, it provides a seamless, responsive interface for users to discover, create, and share recipes.

### 🌟 What Makes RecipeEasy Special?

- **Intelligent AI Selection**: Automatically chooses the best AI model based on language and content
- **Multi-Language Support**: Native Chinese and English support with localized AI models
- **Smart Image Generation**: AI-powered recipe visualization with context-aware model selection
- **Modern Architecture**: Built on Cloudflare's edge computing for global performance
- **User-Centric Design**: Personalized experience with credit system and user preferences

## ✨ Key Features

### 🤖 AI-Powered Generation
- **Smart Model Selection**: Automatically selects optimal AI models based on language
- **Multi-Language AI**: Chinese (Qwen Plus) and English (GPT-4o Mini) support
- **Context-Aware Responses**: AI understands cooking context and dietary preferences

### 🎨 Visual Experience
- **AI Image Generation**: High-quality recipe images generated by specialized AI models
- **Smart Model Routing**: Automatically selects Wanx (Chinese) or Flux Schnell (English)
- **Responsive Design**: Perfect experience across all devices and screen sizes

### 🔐 User Management
- **Authentication**: Google OAuth and email-based login systems
- **Credit System**: Free credits for new users with fair usage policies
- **Personalization**: User preferences and recipe history tracking

### 🌙 User Experience
- **Theme Support**: Light and dark mode with system preference detection
- **Internationalization**: Full i18n support with automatic language detection
- **Performance**: Optimized for speed with edge computing and CDN delivery

## 🚀 Quick Start

### 📋 Prerequisites

- **Node.js**: 18.0.0 or higher
- **npm**: 9.0.0 or higher (or yarn)
- **Git**: Latest version
- **Cloudflare Account**: For backend services

### 🛠️ Installation

```bash
# 1. Clone the repository
git clone https://github.com/nasirannn/recipe-easy.git
cd recipe-easy

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env.local

# 4. Configure your environment variables
# Edit .env.local with your API keys and configuration

# 5. Start development server
npm run dev
```

### 🌐 Access Your Application

- **Frontend**: [http://localhost:3000](http://localhost:3000)
- **Backend API**: [http://localhost:8787](http://localhost:8787) (when running locally)

## 🏗️ Architecture

RecipeEasy follows a modern, scalable architecture pattern:

```
┌─────────────────┐    ┌──────────────────────┐    ┌─────────────────┐
│   Frontend      │    │    Backend API       │    │   Data Layer    │
│   (Next.js 14)  │◄──►│ (Cloudflare Workers)│◄──►│ (D1 + R2)       │
│                 │    │                      │    │                 │
│ • React App     │    │ • Serverless API    │    │ • SQLite DB     │
│ • TypeScript    │    │ • AI Integration     │    │ • Object Storage│
│ • Tailwind CSS  │    │ • File Management   │    │ • Edge Caching  │
│ • Responsive UI │    │ • Authentication     │    │ • Global CDN    │
└─────────────────┘    └──────────────────────┘    └─────────────────┘
```

### 🔄 Data Flow

1. **User Request** → Frontend (Next.js)
2. **API Call** → Cloudflare Worker
3. **AI Processing** → External AI Services
4. **Data Storage** → D1 Database + R2 Storage
5. **Response** → User Interface

## ⚙️ Configuration

### 🔑 Environment Variables

Create a `.env.local` file with the following configuration:

```bash
# ===== Supabase Configuration =====
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

# ===== Cloudflare Worker =====
WORKER_URL=https://your-worker.your-domain.workers.dev

# ===== AI Service API Keys =====
QWENPLUS_API_KEY=your_qwenplus_api_key
DASHSCOPE_API_KEY=your_dashscope_api_key
REPLICATE_API_TOKEN=your_replicate_api_token

# ===== Analytics (Optional) =====
NEXT_PUBLIC_GA_ID=your_google_analytics_id
NEXT_PUBLIC_CLARITY_ID=your_microsoft_clarity_id
```

### 🔧 Configuration Files

- **`wrangler.toml`**: Cloudflare Workers configuration
- **`next.config.js`**: Next.js configuration
- **`tailwind.config.js`**: Tailwind CSS configuration
- **`tsconfig.json`**: TypeScript configuration
